## 🔧 ПРИЧИНА

Функция `recalculate_participant_fields` **добавляла 1 месяц** за каждый оплаченный:

```python
for _ in range(months_paid):  # 1 итерация
    start_month += 1  # Март (03) → Апрель (04) ❌
```

**Результат:** Март + 1 = Апрель, но это **неправильно**!

## ✅ ИСПРАВЛЕНИЕ

`paid_until_month` должен показывать **ПОСЛЕДНИЙ оплаченный месяц** (включительно), а не следующий.

**Новая логика:**
```python
# Отсчитываем (months_paid - 1) месяцев вперёд
# Т.к. первый месяц уже оплачен (start_date)
for _ in range(months_paid - 1):  # 0 итераций при months_paid=1
    start_month += 1

# paid_until_month = ПОСЛЕДНИЙ оплаченный месяц
participant.paid_until_month = f"{start_year}-{start_month:02d}"
```

**Результат:**
- months_paid = 1 → цикл 0 раз → `paid_until_month = 2026-03` ✅
- months_paid = 2 → цикл 1 раз → `paid_until_month = 2026-04` ✅
