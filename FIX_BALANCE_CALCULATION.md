# ✅ ИСПРАВЛЕНИЕ РАСЧЕТА БАЛАНСА УЧАСТНИКОВ

**Дата:** 11.03.2026  
**Статус:** ✅ **ИСПРАВЛЕНО**

---

## 🐛 ПРОБЛЕМА

В таблице участников меню "Участники" **неправильно считался баланс**:

**Пример Григорьева В.:**
- Группа: VPN Level-2 (300 руб./мес)
- Внёс: 1500 руб.
- Дата начала: 2026-03
- **Баланс до исправления:** +150 руб. ❌
- **Баланс после исправления:** 1350 руб. ✅

---

## 🔧 ПРИЧИНА

Старая формула расчета баланса:
```python
balance = total_amount % monthly_fee  # остаток от деления
```

**Проблема:**
- Не учитывалось сколько месяцев прошло с даты начала
- Не учитывалась смена групп с разной стоимостью
- Баланс показывал только "копейки" после оплаты полных месяцев

---

## ✅ РЕШЕНИЕ

### 1. Новая формула баланса

```python
balance = total_paid - amount_required
```

Где:
- `total_paid` = сумма всех доходов участника
- `amount_required` = сколько должен заплатить за всё время (с учётом смены групп и пауз)

### 2. Учёт истории членства в группах

`amount_required` считается по таблице `membership_history`:

```python
for membership in memberships:
    group = membership.group
    months_in_group = количество_месяцев_в_этой_группе
    amount_required += months_in_group * group.monthly_fee
```

**Пример:**
- Март-Июль: 5 месяцев × 150 руб. = 750 руб.
- Август-Сентябрь: 2 месяца × 300 руб. = 600 руб.
- **Итого должен:** 1350 руб.

### 3. Защита от дублирования месяцев

Если в `membership_history` несколько записей с одинаковыми месяцами, они учитываются только один раз:

```python
processed_months = set()  # Track months already counted

for membership in memberships:
    for month in period:
        if month not in processed_months:
            processed_months.add(month)
            months_in_group += 1
```

### 4. Добавлены поля паузы

В таблицу `participants` добавлены поля:
- `paused_from` — с какого месяца пауза (формат "YYYY-MM")
- `paused_to` — по какой месяц пауза включительно (формат "YYYY-MM")

**Месяцы паузы вычитаются из `amount_required`.**

## 📝 ФОРМУЛЫ

### Баланс
```
balance = total_paid - amount_required
```

### amount_required
```
amount_required = Σ(monthly_fee_группы × месяцев_в_группе) - месяцев_паузы
```

### paid_until_month
```
months_paid = int(total_paid / current_monthly_fee)
paid_until_month = start_date + months_paid - 1
```

## 🧪 ПРИМЕРЫ РАСЧЕТА

### Пример 1: Григорьев В.
```
Группа: VPN Level-1 (150 руб./мес)
Дата начала: 2026-03
Сейчас: 2026-03
Внёс: 1500 руб.

amount_required = 1 × 150 = 150 руб. (март)
balance = 1500 - 150 = 1350 руб. ✅
months_paid = 1500 / 150 = 10 месяцев
paid_until_month = 2026-03 + 10 - 1 = 2026-12 ✅
```

### Пример 2: Смена группы
```
Группа: VPN Level-2 (300 руб./мес)
Дата начала: 2026-03
Сейчас: 2026-06
Внёс: 1500 руб.

История:
- Март-Май: 3 месяца × 150 руб. = 450 руб.
- Июнь: 1 месяц × 300 руб. = 300 руб.

amount_required = 450 + 300 = 750 руб.
balance = 1500 - 750 = 750 руб. ✅
```

### Пример 3: С паузой
```
Группа: VPN Level-1 (150 руб./мес)
Дата начала: 2026-03
Сейчас: 2026-06
Пауза: 2026-04 - 2026-05
Внёс: 1500 руб.

История:
- Март-Июнь: 4 месяца × 150 руб. = 600 руб.
- Пауза (Апрель-Май): -2 месяца × 150 руб. = -300 руб.

amount_required = 600 - 300 = 300 руб. (только март и июнь)
balance = 1500 - 300 = 1200 руб. ✅
```
